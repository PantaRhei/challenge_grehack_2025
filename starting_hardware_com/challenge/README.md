# Objective

The challenge aims to reveal three major hardware communication protocols.
A logic-analyzer capture was recorded for each of the following:

* a UART communication
* an I²C communication
* an SPI communication

Each capture provides one part of the flag. The complete flag is the concatenation of the three parts.
Flag format: `GF{FLAG}`. Order of parts: **UART**, then **I2C**, then **SPI**. (There are no traps.)

# Provided files

Three capture files recorded with PulseView are provided.
Each file corresponds to one of the communication protocols above.

# General information

Captures were taken on programmable boards, including an **Arduino MKR WiFi 1010**, an **ESP32**, and an **STM32L476RG**.
